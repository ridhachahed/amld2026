Architecture

RAG Agent Architecture

The agent follows a standard retrieval-augmented generation pattern, enhanced with a tool-calling loop. When a user asks a question, the agent first determines if it needs to call a tool (SQL query, API call) or retrieve context from the vector store, then generates a response with the LLM.

Step 1

Ingest Documents & Generate Embeddings

The first step is to populate the vector store. HeatWave's Auto Parallel Load handles the heavy lifting: it discovers documents in OCI Object Storage, chunks them, generates embeddings using the configured model, and stores everything in a queryable vector store table.

-- Load documents from Object Storage into the vector store -- Auto Parallel Load handles chunking + embedding generation CALL sys.heatwave_load( JSON_ARRAY('amld_workshop', 'vector_store'), JSON_OBJECT( 'mode', 'normal', 'policy', 'enable_auto', 'output', 'normal' ) );

What happens behind the scenes: Auto Parallel Load reads files from the OCI Object Storage bucket, splits documents into semantically meaningful chunks, generates vector embeddings for each chunk using the configured model (default: multilingual-e5-small), creates a VECTOR column with automatic HNSW indexing, and loads everything into the HeatWave cluster for fast retrieval.

Verify the ingested data

-- Check what was loaded SELECT COUNT(*) AS total_chunks, COUNT(DISTINCT doc_name) AS total_docs FROM amld_workshop.vector_store; -- Preview chunks from a specific document SELECT doc_name, segment_id, chunk_text FROM amld_workshop.vector_store WHERE doc_name LIKE '%report%' LIMIT 5;
Step 2

Retrieve Context & Generate Answers

Now that documents are in the vector store, we can build the RAG pipeline. The simplest approach uses the built-in ML_RAG() routine which handles retrieval and generation in a single call. For more control, you can build a custom pipeline.

Option A: Built-in ML_RAG (one-step)

-- One-step RAG: retrieve + generate in a single call CALL sys.ML_RAG( 'What were the key findings from the Q3 analysis?', @answer, JSON_OBJECT( 'n_citations', 3, 'model_id', 'llama3.2-3b-instruct-v1', 'tables', JSON_ARRAY('amld_workshop.vector_store') ) ); SELECT JSON_PRETTY(@answer) AS rag_response\G

Option B: Custom RAG pipeline (more control)

-- Step 1: Embed the user query SET @q_vec = (SELECT ML_EMBED_ROW( 'What were the key findings from the Q3 analysis?', 'multilingual-e5-small' )); -- Step 2: Retrieve top-K context chunks SELECT chunk_text, doc_name, DISTANCE(embedding, @q_vec, 'COSINE') AS score INTO @ctx FROM amld_workshop.vector_store ORDER BY score LIMIT 5; -- Step 3: Build augmented prompt and generate CALL sys.ML_GENERATE( CONCAT( 'Based on the following context, answer the question.\n\nContext:\n', @ctx, '\n\nQuestion: What were the key findings from the Q3 analysis?' ), @output );

When to use each option: Use ML_RAG() for standard Q&A where the built-in pipeline is sufficient. Use the custom pipeline when you need to pre-filter documents, add business logic between retrieval and generation, or combine context from multiple sources (vector store + SQL queries + API responses).

Step 3

Wire in Custom Tools

A RAG chatbot that only answers questions from documents is limited. By adding custom tools, the agent can query structured data, perform calculations, call APIs, and take actions. Here's how to implement tool-calling in the Python agent.

Define tool functions

import mysql.connector import json # Tool 1: Query structured data via SQL def query_products(category: str, limit: int = 5) -> dict: """Search the product catalog by category.""" cursor.execute( "SELECT name, price, stock FROM products " "WHERE category = %s ORDER BY price DESC LIMIT %s", (category, limit) ) return {"products": cursor.fetchall()} # Tool 2: RAG search over documents def search_documents(query: str, n_results: int = 3) -> dict: """Search unstructured documents using vector similarity.""" cursor.callproc('sys.ML_RAG', [query, '@out', json.dumps({ "n_citations": n_results, "tables": ["amld_workshop.vector_store"] })]) cursor.execute("SELECT @out") return json.loads(cursor.fetchone()[0]) # Tool 3: Natural language SQL def ask_database(question: str) -> dict: """Convert natural language to SQL and execute.""" cursor.callproc('sys.NL_SQL', [question, '@sql']) cursor.execute("SELECT @sql") sql = cursor.fetchone()[0] cursor.execute(sql) return {"sql": sql, "results": cursor.fetchall()}

Tool registry & routing

# Register tools with descriptions for the agent TOOLS = { "query_products": { "fn": query_products, "description": "Search the product catalog by category", "params": {"category": "str", "limit": "int"} }, "search_documents": { "fn": search_documents, "description": "Search documents using RAG", "params": {"query": "str", "n_results": "int"} }, "ask_database": { "fn": ask_database, "description": "Ask a question in natural language about structured data", "params": {"question": "str"} } }
Advanced

Model Context Protocol (MCP) Integration

For production-grade agent architectures, MySQL HeatWave provides an official MCP Server that exposes database capabilities as standardized tools for any MCP-compatible AI client (Claude Desktop, VS Code, Cline, etc.).

Fig 8 ‚Äî MCP Server Architecture

The MCP Server provides tools for connection management, SQL execution, schema browsing, vector store operations (including ingestion from Object Storage), RAG queries, and NL2SQL. It supports both homogeneous and heterogeneous database connections, meaning you can mix MySQL HeatWave (cloud) and MySQL AI (on-premise) endpoints.

Step 4

Putting It All Together: The Agent Loop

Here's the complete agent loop that ties ingestion, RAG, and tool calling together into a conversational interface:

import mysql.connector import json class RAGAgent: def __init__(self, connection_config, tools): self.conn = mysql.connector.connect(**connection_config) self.cursor = self.conn.cursor(dictionary=True) self.tools = tools self.history = [] def classify_intent(self, user_msg): """Determine if query needs RAG, SQL, or a custom tool.""" self.cursor.callproc('sys.ML_GENERATE', [ json.dumps({"prompt": f"""Classify this query into one of: - rag: needs document search - sql: needs structured data query - tool: needs a specific action Query: {user_msg} Respond with just the category."""}), '@intent' ]) self.cursor.execute("SELECT @intent") return self.cursor.fetchone()['@intent'].strip() def chat(self, user_msg): """Main agent loop ‚Äî classify, route, respond.""" intent = self.classify_intent(user_msg) if intent == 'rag': self.cursor.callproc('sys.ML_RAG', [ user_msg, '@answer', json.dumps({"n_citations": 3}) ]) self.cursor.execute("SELECT @answer") response = self.cursor.fetchone()['@answer'] elif intent == 'sql': result = self.tools['ask_database']['fn'](user_msg) response = f"SQL: {result['sql']}\nResults: {result['results']}" else: response = self.route_to_tool(user_msg) self.history.append({"user": user_msg, "assistant": response}) return response
